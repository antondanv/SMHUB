## Сущности

### users

Пользователи системы.

Поля:

- `id` — уникальный идентификатор пользователя.
- `email` — почта для входа и связи.
- `username` — уникальное имя пользователя.
- `hashed_password` — пароль в зашифрованном виде.
- `full_name` — ФИО или имя студента.
- `role` — роль в системе: студент, модератор, администратор.
- `is_active` — активен ли аккаунт.
- `course_id` — курс пользователя.
- `program_id` — направление или специальность пользователя.
- `group_name` — учебная группа.
- `created_at` — дата создания аккаунта.

---

### courses

Курсы обучения.

Поля:

- `id` — уникальный идентификатор курса.
- `name` — название курса, например `1 курс`.
- `number` — числовое значение курса, например `1`, `2`, `3`.

---

### programs

Направления или специальности.

Поля:

- `id` — уникальный идентификатор направления.
- `name` — название специальности.
- `code` — официальный код специальности.
- `description` — краткое описание.

---

### subjects

Учебные предметы.

Поля:

- `id` — уникальный идентификатор предмета.
- `name` — название предмета.
- `description` — краткое описание предмета.

---

### subject_programs

Связь предметов с направлениями и курсами.

Поля:

- `id` — уникальный идентификатор записи.
- `subject_id` — предмет.
- `program_id` — направление.
- `course_id` — курс.

Назначение:

- позволяет определить, какие предметы относятся к какому направлению и курсу;
- нужна для персонализации и подборок.

---

### material_types

Типы учебных материалов.

Поля:

- `id` — уникальный идентификатор типа.
- `name` — название типа материала.

Примеры:

- лекция
- конспект
- лабораторная
- презентация
- тест
- методичка

---

### materials

Основная сущность проекта — учебные материалы.

Поля:

- `id` — уникальный идентификатор материала.
- `title` — название материала.
- `description` — описание материала.
- `author_id` — пользователь, который загрузил материал.
- `subject_id` — предмет, к которому относится материал.
- `material_type_id` — тип материала.
- `course_id` — курс, для которого материал наиболее подходит.
- `program_id` — направление, для которого материал наиболее подходит.
- `file_url` — путь или ссылка на файл.
- `file_name` — имя файла.
- `file_size` — размер файла.
- `mime_type` — тип файла, например `application/pdf`.
- `status` — статус материала.
- `views_count` — количество просмотров.
- `downloads_count` — количество скачиваний.
- `likes_count` — количество лайков.
- `comments_count` — количество комментариев.
- `favorites_count` — сколько раз добавили в избранное.
- `published_at` — дата публикации.
- `created_at` — дата создания.
- `updated_at` — дата обновления.

Назначение:

- это центральная сущность всей системы;
- именно вокруг нее строится поиск, каталог, комментарии, рейтинг и персонализация.

---

### comments

Комментарии к материалам.

Поля:

- `id` — уникальный идентификатор комментария.
- `material_id` — материал, к которому оставлен комментарий.
- `user_id` — автор комментария.
- `content` — текст комментария.
- `created_at` — дата создания.
- `updated_at` — дата изменения.

Назначение:

- позволяет обсуждать материалы;
- дает обратную связь и повышает полезность контента.

---

### likes

Лайки материалов.

Поля:

- `id` — уникальный идентификатор лайка.
- `user_id` — кто поставил лайк.
- `material_id` — какому материалу поставлен лайк.
- `created_at` — дата создания лайка.

Назначение:

- помогает понять, какие материалы нравятся пользователям;
- используется в популярности материалов.

---

### favorites

Избранные материалы пользователя.

Поля:

- `id` — уникальный идентификатор записи.
- `user_id` — пользователь.
- `material_id` — материал, добавленный в избранное.
- `created_at` — дата добавления.

Назначение:

- позволяет быстро возвращаться к полезным материалам;
- формирует личную подборку пользователя.


---

## Связи

- `users.course_id → courses.id` — пользователь относится к определенному курсу.
- `users.program_id → programs.id` — пользователь относится к определенному направлению.
- `subject_programs.subject_id → subjects.id` — запись ссылается на предмет.
- `subject_programs.program_id → programs.id` — запись ссылается на направление.
- `subject_programs.course_id → courses.id` — запись ссылается на курс.
- `materials.author_id → users.id` — материал загружен конкретным пользователем.
- `materials.subject_id → subjects.id` — материал относится к предмету.
- `materials.material_type_id → material_types.id` — материал имеет определенный тип.
- `materials.course_id → courses.id` — материал связан с курсом.
- `materials.program_id → programs.id` — материал связан с направлением.
- `comments.material_id → materials.id` — комментарий относится к материалу.
- `comments.user_id → users.id` — комментарий написан пользователем.
- `likes.user_id → users.id` — лайк поставлен пользователем.
- `likes.material_id → materials.id` — лайк относится к материалу.
- `favorites.user_id → users.id` — избранное принадлежит пользователю.
- `favorites.material_id → materials.id` — в избранное добавлен материал.
- `ratings.user_id → users.id` — оценка поставлена пользователем.
- `ratings.material_id → materials.id` — оценка относится к материалу.

## Ограничения

### users

- `email unique` — один email нельзя использовать в нескольких аккаунтах.
- `username unique` — имя пользователя должно быть уникальным.

### likes

- `unique(user_id, material_id)` — один пользователь может поставить только один лайк одному материалу.

### favorites

- `unique(user_id, material_id)` — один материал нельзя добавить в избранное повторно одним и тем же пользователем.

### ratings

- `unique(user_id, material_id)` — один пользователь может хранить только одну оценку на один материал.

### subject_programs

- `unique(subject_id, program_id, course_id)` — нельзя дважды создать одну и ту же связь предмета, направления и курса.

---

## Роли

- `student` — обычный пользователь, который ищет, загружает, скачивает и оценивает материалы.
- `moderator` — пользователь, который может проверять и модерировать материалы и комментарии.
- `admin` — полный доступ к системе, пользователям и справочникам.

---

## Статусы materials

- `draft` — черновик, еще не опубликован.
- `pending` — ожидает проверки или публикации.
- `published` — опубликован и доступен всем.
- `rejected` — отклонен.
- `archived` — убран из активного каталога, но не удален окончательно.

---

## Главная страница

### Для неавторизованного пользователя

- **последние материалы** — показывает новые публикации.
- **популярные материалы** — показывает востребованные материалы.
- **материалы по предметам** — помогает быстро перейти к контенту по нужной дисциплине.

### Для авторизованного пользователя

- **материалы его курса** — материалы, релевантные курсу студента.
- **материалы его направления** — материалы по специальности пользователя.
- **новые материалы по связанным предметам** — свежие материалы по тем предметам, которые связаны с курсом и направлением студента.
- **популярные материалы среди студентов того же курса** — материалы, которые особенно востребованы у похожих пользователей.

## Основные API

### auth

- `POST /auth/register` — регистрация нового пользователя.
- `POST /auth/login` — вход в систему и получение токена.
- `GET /auth/me` — получение данных текущего авторизованного пользователя.

### users

- `GET /users/me` — просмотр своего профиля.
- `PATCH /users/me` — редактирование своего профиля.

### materials

- `GET /materials` — получение списка материалов с поиском, фильтрами и сортировкой.
- `POST /materials` — загрузка нового материала.
- `GET /materials/{id}` — просмотр одного материала.
- `PATCH /materials/{id}` — редактирование материала.
- `DELETE /materials/{id}` — удаление материала.

### comments

- `GET /materials/{id}/comments` — список комментариев материала.
- `POST /materials/{id}/comments` — добавление комментария.

### likes

- `POST /materials/{id}/like` — поставить лайк материалу.
- `DELETE /materials/{id}/like` — убрать лайк.

### favorites

- `POST /materials/{id}/favorite` — добавить материал в избранное.
- `DELETE /materials/{id}/favorite` — убрать материал из избранного.
- `GET /users/me/favorites` — получить список избранных материалов текущего пользователя.

### ratings

- `POST /materials/{id}/rating` — поставить оценку материалу.
- `PATCH /materials/{id}/rating` — изменить свою оценку.

### homepage

- `GET /homepage` — получить данные для главной страницы с подборками и блоками персонализации.

## Фильтры каталога

- `search` — поиск по названию материала.
- `subject_id` — фильтр по предмету.
- `material_type_id` — фильтр по типу материала.
- `course_id` — фильтр по курсу.
- `program_id` — фильтр по направлению.
- `sort=new` — сортировка по новизне.
- `sort=popular` — сортировка по популярности.

## Минимальный MVP

- **регистрация** — создание аккаунта.
- **вход** — авторизация пользователя.
- **профиль пользователя** — хранение курса, направления и данных студента.
- **загрузка материалов** — возможность добавить учебный файл.
- **каталог материалов** — общий список всех опубликованных материалов.
- **поиск по названию** — быстрый поиск нужного контента.
- **фильтр по предмету** — отбор материалов по дисциплине.
- **фильтр по типу** — отбор по типу учебного материала.
- **страница материала** — отдельная страница с описанием, файлом, оценками и комментариями.
- **скачивание файла** — получение материала пользователем.
- **комментарии** — обсуждение и обратная связь.
- **лайки** — быстрый способ отметить полезность.
- **рейтинг** — оценка качества материала.
- **избранное** — личное сохранение полезных материалов.
- **главная страница с персонализацией** — подборки по курсу, направлению и популярности.
