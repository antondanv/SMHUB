## Финальная формулировка
Это веб-приложение с frontend на React, backend на FastAPI и базой данных PostgreSQL для хранения, поиска и обмена учебными материалами студентов колледжа.

Пользователи смогут:
- регистрироваться и входить в систему;
- загружать учебные материалы;
- просматривать каталог материалов;
- искать материалы по названию;
- фильтровать их по предмету и типу;
- скачивать файлы;
- оставлять комментарии;
- ставить лайки;
- оценивать материалы;
- добавлять материалы в избранное.

## Позиционирование проекта
Проект позиционируется не как социальная сеть для студентов и не как обычный файлообменник, а как информационная система обмена учебными материалами.

## Основной фокус
Главная цель первой версии — дать студенту быстрый доступ к полезным материалам по его учебным дисциплинам в одном месте.

## Что особенно важно для моего проекта
Так как проект делается в рамках колледжа, на главной странице должна быть логика персонализации.

### Главная страница должна показывать
1. Недавно опубликованные материалы.
2. Популярные материалы.
3. Материалы по моему курсу.
4. Материалы по моему направлению или специальности.
5. Подборку для подготовки по подходящим предметам.

### Базовая логика рекомендаций
Для неавторизованного пользователя:
- последние материалы;
- популярные материалы;
- материалы по предметам.

Для авторизованного пользователя:
- материалы его курса;
- материалы его направления;
- новые материалы по связанным предметам;
- популярные материалы среди студентов того же курса.

## Почему это хороший вариант
- выглядит современно;
- полезно для студентов;
- подходит под тему колледжа;
- не требует сложной рекомендательной системы;
- реально реализуется на связке React + FastAPI + PostgreSQL в рамках [[02_MVP|MVP]].

## Что не входит в первую зафиксированную версию
Чтобы не перегружать проект, в первую версию не включаются:
- нейросетевые рекомендации;
- чат;
- подписки;
- уведомления;
- Telegram-бот;
- подборки пользователей как отдельная большая сущность.

## Итого
Я делаю веб-приложение для колледжа, где React отвечает за пользовательский интерфейс, FastAPI за бизнес-логику и API, а PostgreSQL за хранение данных. В системе студенты смогут загружать и находить учебные материалы, комментировать их, ставить лайки, оценивать и добавлять в избранное, а главная страница помогает быстрее находить полезный контент за счёт ленты новых и популярных материалов, а также простой персонализации по курсу и направлению.

## Ключевая ценность
Сервис экономит время студентов, упрощает подготовку к занятиям и экзаменам и собирает учебные материалы в одном структурированном пространстве.
